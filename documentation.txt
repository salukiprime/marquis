Документация по проекту Marquis Automobile

1. Обзор проекта
Проект представляет собой интерактивное веб-приложение (на чистом JS без фреймворков), разработанное для презентации премиального автомобильного бренда "Marquis Automobile", а также для демонстрации функционала пользовательского "гаража" (La Collection) с сохранением состояния через `localStorage`. 
Архитектура приложения опирается на модульный подход, в котором файлы HTML разделены по смыслу страниц, стили собраны в единый CSS, а JavaScript-логика распределена между несколькими скриптами для обработки локализации, хранилища и рендера.

2. Структура HTML 
Система состоит из четырех основных страниц:
- `index.html` (Le Garage) – Главная страница приложения, которая совмещает презентационный слой (секция Hero с баннером и сетка L'Art de Détail) и персональный "гараж" пользователя. Гараж отображает массив выбранных и сконфигурированных пользователем автомобилей в виде карточек. 
- `catalog.html` (La Collection) – Страница, отображающая каталог доступных базовых моделей (Pavillon, Belvédère, Mistral). При клике на конкретную модель происходит переход к ее детальной конфигурации (L'Atelier).
- `details.html` (L'Atelier) – Страница описания конкретной модели и ее конфигуратор формы. Применяет GET-параметры в URL (например, `?model=pavillon`) для определения контекста.
- `maison.html` (La Maison) – Информационная страница об истории и манифесте бренда Marquis Automobiles с сеткой текстовых блоков и изображений.

Все HTML-файлы имеют общую структуру:
- Верхняя мини-навигация с переключателем языка.
- Главное меню (header) со ссылками на страницы (активная страница выделена).
- Тег `<main>`, куда динамически генерируется содержимое через JS.
- Атрибуты `data-i18n="ключ"` на большинстве элементов для поддержки двуязычности (Английский / Русский).

3. Оформление (CSS)
Файл `css/style.css` является единым источником стилей для всего проекта:
- Опирается на CSS Variables (`:root`) для соблюдения строгой палитры бренда (черный `--color-black`, белый, кремовый `--color-creme`, алый `--color-scarlet`).
- Использует CSS Flexbox и Grid для адаптивного позиционирования элементов (карт гаража, каталога, секций L'Art de Détail).
- Содержит hover-эффекты и плавные переходы (`transition`) для кнопок, карточек машин и чекбоксов (переключателей статуса).

4. Логика (JavaScript)

4.1 Вспомогательные функции (`js/utils.js`)
- `generateId()`: Генерирует уникальные ID для каждой новой созданной карточки автомобиля. Использует `crypto.randomUUID()`, а также имеет механизм обратной совместимости через `Date.now()` и генерацию случайных чисел.
- `formatDate()`: Форматирует сырую дату JS в удобочитаемую (Месяц День, Год) для отображения.

4.2 Хранение данных (`js/storage.js`)
Этот модуль отвечает за взаимодействие с LocalStorage для хранения пользовательского прогресса.
Ключ базы сохраняется в переменной `STORAGE_KEY`.
- `getCollection()`: Запрашивает массив объектов машин. Если `localStorage` пуст, инициализирует массив данными по умолчанию (фиктивные записи Pavillon и Belvédère) и возвращает их.
- `saveCollection(collection)`: Функция-помощник для трансформации массива в строку JSON и записи в хранилище.
- `addVehicle(vehicleData)`: Создает новый объект машины, формируя ей `id` (через `utils.js`), статус 'active', дату, и объединяя пришедшие данные из формы. Обновляет общий список.
- `toggleVehicleStatus(id)`: Изменяет статус машины с 'active' на 'stored' и наоборот, сохраняя изменения.
- `deleteVehicle(id)`: Ищет элемент по его `id`, отфильтровывает массив, удаляя нужную машину.

4.3 Система локализации (`js/lang.js`)
Модуль для перевода интерфейса сайта.
- Объект `translations`: Содержит все переводы в структуре {'Ключ': {en: 'Текст', ru: 'Текст'}}.
- `setLanguage(langCode)`: Записывает в `localStorage` язык пользователя (en или ru) и делает полную перезагрузку страницы.
- `getCurrentLang()`: Возвращает язык текущей сессии.
- `t(key)`: Принимает строковой ключ и отдает значение на нужном языке из массива `translations`.
- `applyTranslations()`: Главная функция, запускается автоматически при `DOMContentLoaded`. Ищет все элементы на HTML-странице с атрибутом `data-i18n` и перезаписывает их содержимое (`innerHTML` или `placeholder`) переведенным текстом. Управляет подсветкой активного языка в верхнем меню.

4.4 Страница Каталога (`js/catalog.js`)
Скрипт рендера списка машин.
- Хранит константный массив данных о доступных моделях: id, название, ключи для перевода.
- Ищет `#catalog-grid` и генерирует набор DOM-узлов (карточек) для каждой модели, устанавливая название, перевод описания из файла языка и ссылку вида `details.html?model={id}`, направляя пользователя в конфигуратор выбранной машины.

4.5 Страница Гаража (`js/main.js`)
Скрипт главного "пульта управления" на главной странице.
- `renderCollection()`: Запрашивает коллекцию у `storage.js`. Если машин нет, выводит информативное сообщение. Если они есть, очищает DOM-дерево сетки гаража и вставляет новые HTML элементы. Извлекает параметры (Двигатель, Цвет, Статус машины).
- `attachEventListeners()`: Подключает интерактивность к свежесозданным карточкам. 
   1. Чекбоксы переключают статус (active/stored) и обновляют подпись; 
   2. Кнопки удаления `btn-delete` вызывают окно `confirm` для подтверждения продажи (удаления) машины из базы. При положительном ответе удаляет из хранилища и перерисовывает коллекцию.

4.6 Детальное описание и Конфигуратор (`js/details.js`)
Это самый сложный по логике файл, связывающий отображение информации о машине с формой создания.
- Объект `modelData`: Огромная структура, хранящая ключи, специфичные для каждой модели: фоновое изображение, вариации доступных двигателей, и вложенный объект спецификаций. Спецификации зависят от комбинации "Год выпуска + Двигатель".
- `DOMContentLoaded`: Извлекает имя модели из параметра URL. Выводит заглушку "404", если машина не найдена.
- Рендеринг страницы: Динамически формирует секцию с фотографиями (выбирает сетку для Mistral, или большое фото для Pavillon / Belvédère).
- Заполнение формы (L'Atelier): Динамически внедряет опции двигателей и цветов (`<option>`) в HTML-теги формы.
- Интерактивность спецификаций (`updateSpecs()`): При изменении года или типа двигателя, выводит специальный блок "Ключевые характеристики" (разгон, скорость, цена), забирая свойства из `modelData` для нужной комбинации параметров.
- Валидация и Сохранение: При событии `submit` формы, происходит превентивная остановка `e.preventDefault()`. Проверяется наличие выбранных параметров (валидация). Если все верно, данные заворачиваются в объект, вызывается функция `addVehicle` из `storage.js`, выводится всплывающее окно и происходит редирект на 1 страницу (`index.html`) в гараж.
